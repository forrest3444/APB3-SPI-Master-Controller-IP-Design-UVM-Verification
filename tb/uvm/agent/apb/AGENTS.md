# APB Agent

## Objective
This directory implements the APB master agent, used for configuring the DUT, accessing registers, and performing read/write operations on the FIFO data ports.

## What Files in This Directory Should Do
- Implement `apb_agent`
- Implement `apb_driver`
- Implement `apb_monitor`
- Implement `apb_sequencer`

## Functional Boundary
The APB agent is only concerned with APB protocol behavior and APB transactions; it does **not** handle the correctness of SPI timing or data results.

## Mandatory Considerations
- Adhere to the two-phase APB timing (setup and access).
- Comply with the `always-ready` semantics.
- Model read and write accesses uniformly.
- The monitor must reconstruct complete APB transactions.
- Driver behavior must be strictly protocol-compliant; avoid direct "magic register writes".

## Exclusions
- Do not check SPI data directly within the APB driver.
- Do not derive frame-level results within the APB monitor.
- Do not embed scoreboard logic within the agent.

## Design Requirements
- The APB agent serves as the **only official stimulus entry point** for the control plane.
- Transactions generated by the APB monitor must be sufficient for coverage collection and scoreboard comparison.
