# Coverage Waivers

This directory stores approved coverage waivers used for coverage closure.
Waivers are engineering exceptions, not a substitute for missing tests.

## Policy

Use this flow for every waiver:

1. Run a clean coverage regression and generate the raw merged coverage
   database/report without waivers.
2. Review the raw merged coverage database in DVE.
3. Confirm the hole is unreachable, illegal, defensive-only, or not a project
   coverage target according to the RTL spec and verification plan.
4. Add or update a `VTR-*` entry in `tb/doc/COVERAGE_VTRACK.md` and
   `tb/doc/COVERAGE_VTRACK_CN.md`.
5. Add the waiver rationale to the source waiver file in this directory.
6. Exclude the reviewed item in DVE and export the tool-generated `.el` file.
7. Re-open the same merged coverage database in DVE with the exported `.el`
   file loaded.
8. Re-review the waiver when the RTL, spec, or coverage model changes.

## Files

| File | Purpose |
| --- | --- |
| `code_coverage_waivers.yml` | Human-reviewable source of truth for code coverage waivers. |
| `dve_waivers.el` | DVE-exported waiver file loaded by DVE after manual review. This file is expected to be generated by DVE, not hand-written. |

## Makefile Integration

Generate raw coverage first:

```sh
make -C tb cov_regression BUILD_NAME=regression_cov REGRESSION_RUN_TIME=10s
```

Open the raw merged database for manual DVE review:

```sh
make -C tb view_cov MERGED_COV_DIR=sim/merged_cov.vdb
```

After DVE exports the approved `.el` file to `tb/cov_waivers/dve_waivers.el`,
re-open the same merged database with the waiver file loaded:

```sh
make -C tb view_cov \
  MERGED_COV_DIR=sim/merged_cov.vdb \
  COV_WAIVER_EL=./cov_waivers/dve_waivers.el
```

The merged database remains the single coverage database. The `.el` file is
applied only when DVE opens it for review.
